# TASK-002: 読込み状況管理システム - 詳細要件定義

## タスク概要

セッション内でのドキュメント読込み状況を記録・管理し、構成変更時の自動リセット機能を提供するシステムの実装。

## 詳細要件

### 機能要件

#### FR-004: 読込み状況記録機能
- システムは各ドキュメントの読込み状況を一時ファイルに記録できること
- 記録は`/tmp/.claude_docs_read_status_$$`形式でプロセスID付きで管理すること
- 読込み済みドキュメントのファイルパスと読込み時刻を記録すること

#### FR-005: 読込み状況取得機能  
- システムは記録された読込み状況を取得・確認できること
- 特定ドキュメントの読込み状況を個別に確認できること
- 全ドキュメントの読込み状況を一覧で確認できること

#### FR-006: 自動リセット機能
- システムは構成変更検知時に読込み状況を自動リセットできること
- セッション終了時に一時ファイルを自動削除できること
- 強制リセット機能を提供できること

### 非機能要件

#### NFR-003: セキュリティ要件
- 読込み状況ファイルは他ユーザーから読み取り不可であること（権限600）
- セッション終了時に確実にファイルが削除されること
- 一時ファイルは推測困難な名前を使用すること

#### NFR-004: 信頼性要件
- ファイル書き込み失敗時は適切なエラーハンドリングを行うこと
- 並行実行時の競合状態を適切に処理すること
- プロセス異常終了時でもファイルリークを最小限に抑えること

#### NFR-005: 保守性要件
- デバッグ情報を適切にログ出力すること
- エラー状況を分かりやすく報告すること
- 設定可能なパラメータを外部化すること

## 実装詳細

### 関数仕様

#### `record_doc_read_status()`
```bash
# 引数: $1 = ドキュメントファイルパス
# 戻り値: 0 = 成功, 1 = ファイル書き込み失敗, 2 = パラメータエラー
# 機能: ドキュメントの読込み状況を記録
```

#### `check_doc_read_status()`  
```bash
# 引数: $1 = ドキュメントファイルパス
# 戻り値: 0 = 読込み済み, 1 = 未読, 2 = ファイル読み込み失敗
# 機能: 特定ドキュメントの読込み状況を確認
```

#### `get_unread_docs()`
```bash
# 戻り値: 0 = 未読あり, 1 = 全読済み, 2 = エラー
# 出力: 未読ドキュメントの一覧（改行区切り）
# 機能: 未読ドキュメントの一覧を取得
```

#### `reset_read_status()`
```bash
# 戻り値: 0 = 成功, 1 = ファイル削除失敗
# 機能: 読込み状況を強制リセット
```

### エラーハンドリング

#### E-004: 一時ファイル作成エラー
- `/tmp`ディレクトリへの書き込み権限がない場合
- 対処: エラーメッセージを表示し、処理を中断

#### E-005: ファイル読み込みエラー
- 状況ファイルが破損している場合
- 対処: ファイルをリセットして再初期化

#### E-006: 並行アクセスエラー
- 複数プロセスが同時に状況ファイルを更新する場合
- 対処: ファイルロック機能または最終更新時刻による制御

### ファイル形式仕様

#### 状況ファイルフォーマット
```
# claude_docs_read_status - auto generated
# format: filepath|timestamp|hash
docs/rules/frontend.md|1640123456|abc123
docs/rules/styling-rule.md|1640123789|def456
# config_hash: f7f078e508c45697b7991ac048a12f55
```

## テスト要件

### 単体テスト

#### UT-004: 読込み状況記録テスト
- 正常系: ドキュメントの読込み状況が正しく記録される
- 異常系: 権限不足時のエラーハンドリング
- 境界値: 同一ドキュメントの重複記録

#### UT-005: 読込み状況確認テスト  
- 正常系: 記録済みドキュメントが正しく判定される
- 異常系: 状況ファイル破損時の処理
- 境界値: 空の状況ファイル

#### UT-006: 未読ドキュメント取得テスト
- 正常系: 未読ドキュメントが正しく一覧化される
- 異常系: 全ドキュメント読込み済みの場合
- 境界値: ドキュメント数0個の場合

#### UT-007: 状況リセットテスト
- 正常系: 状況ファイルが完全に削除される
- 異常系: ファイル削除権限がない場合
- 境界値: 存在しないファイルのリセット

### 統合テスト

#### IT-002: ワークフロー統合テスト
- シナリオ1: 初回実行 → ドキュメント読込み → 状況記録 → 確認
- シナリオ2: 構成変更 → 自動リセット → 再読込み要求
- シナリオ3: セッション終了 → ファイル自動削除
