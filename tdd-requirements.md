# TASK-203: ルーチン管理画面実装 - 要件定義

## タスク概要
- **タスクID**: TASK-203
- **タスクタイプ**: TDD
- **要件リンク**: REQ-002 (ルーチン作成), REQ-101 (頻度ベース検証)
- **依存タスク**: TASK-102 (ルーチン管理API) ✅, TASK-104 (カテゴリ管理API) ✅

## 実装対象画面
- **メインページ**: `app/routines/page.tsx`
- **コンポーネント**: ルーチン一覧表示、作成・編集フォーム、カテゴリフィルター
- **統合対象**: API エンドポイント (TASK-102, TASK-104で実装済み)

## 機能要件

### 1. ルーチン一覧表示
- **FR-203-01**: 認証されたユーザーの全ルーチンを一覧表示
- **FR-203-02**: 各ルーチンの基本情報表示（名前、頻度、カテゴリ、ステータス）
- **FR-203-03**: アクティブ・非アクティブの視覚的区別
- **FR-203-04**: 実行ボタンの表示（アクティブルーチンのみ）

### 2. ルーチン作成・編集フォーム
- **FR-203-05**: 新規ルーチン作成フォーム
  - 名前 (必須, 1-100文字)
  - カテゴリ (必須, ドロップダウン選択)
  - 頻度タイプ (必須, daily/weekly/monthly/custom)
  - 頻度詳細 (頻度タイプに応じた条件分岐)
  - メモ (任意, 0-500文字)
  - アクティブ状態 (デフォルトtrue)

- **FR-203-06**: 既存ルーチン編集機能
  - 全フィールド編集可能
  - 実行記録が存在する場合の制限事項表示

- **FR-203-07**: リアルタイムバリデーション
  - フィールド単位でのエラー表示
  - フォーム送信前の包括的バリデーション

### 3. 頻度ベース検証機能 (REQ-101)
- **FR-203-08**: 頻度タイプ別UI表示制御
  - daily: 追加フィールドなし
  - weekly: 曜日選択 (複数選択可能)
  - monthly: 日付選択 (複数選択可能)
  - custom: カスタム間隔数値入力

- **FR-203-09**: 頻度検証ロジック統合
  - API連携による頻度バリデーション
  - エラー時の適切なユーザーフィードバック

### 4. カテゴリ管理統合
- **FR-203-10**: カテゴリ一覧の動的取得・表示
- **FR-203-11**: カテゴリフィルター機能
- **FR-203-12**: 新規カテゴリ作成機能（オプション）

### 5. ルーチン操作機能
- **FR-203-13**: ルーチン削除機能（論理削除）
- **FR-203-14**: ルーチン復元機能
- **FR-203-15**: ルーチン実行機能（実行記録作成への誘導）

### 6. 一覧操作・UX機能
- **FR-203-16**: ソート機能（名前、作成日、カテゴリ別）
- **FR-203-17**: 検索・フィルター機能
- **FR-203-18**: ページネーション対応

## 技術仕様・制約

### 1. 技術スタック
- **フロントエンド**: Next.js 15.4.5, React, TypeScript
- **UI Framework**: Tailwind CSS 4.x, 共通UIコンポーネント (TASK-208)
- **状態管理**: React hooks (useState, useReducer), Server Actions
- **フォーム管理**: React Hook Form + Zod validation
- **API通信**: fetch API with error handling

### 2. ファイル構成
- **メインページ**: `src/app/routines/page.tsx`
- **コンポーネント**: `src/components/routines/`
  - `RoutineList.tsx` - ルーチン一覧表示
  - `RoutineForm.tsx` - 作成・編集フォーム
  - `RoutineCard.tsx` - ルーチンカード表示
  - `FrequencySelector.tsx` - 頻度選択コンポーネント
  - `CategoryFilter.tsx` - カテゴリフィルター
- **型定義**: `src/shared/types/` (既存利用)
- **API通信**: `src/lib/api/routines.ts`

### 3. 外部依存
- **API**: TASK-102 (ルーチン管理API), TASK-104 (カテゴリ管理API)
- **共通コンポーネント**: TASK-208 (共通UIコンポーネント)
- **認証**: 認証コンテキスト (TASK-201)
