# TASK-203: ユーザー関連コンポーネント移行 - 要件定義

## 概要
設定関連コンポーネントをユーザードメインに移行し、App-Common-Model アーキテクチャに準拠した構造に再編成する。

## 詳細要件

### 1. コンポーネント移行要件

#### 1.1 移行対象コンポーネント
- `src/components/settings/ProfileSettings.tsx` → `src/model/user/components/profile/ProfileSettings.tsx`
- `src/components/settings/AppSettings.tsx` → `src/model/user/components/settings/AppSettings.tsx`
- 対応するテストファイルも同構造で移行

#### 1.2 アーキテクチャ準拠
- ユーザードメインモデル配下への配置
- `profile/` と `settings/` サブディレクトリでの機能別分類
- 既存の UserAvatar は TASK-201 で既に移行済み（重複回避）

### 2. 機能要件

#### 2.1 ProfileSettings コンポーネント
- **表示機能**: ユーザープロフィール情報の表示
- **編集機能**: プロフィール設定項目の編集フォーム
- **保存機能**: 変更内容の保存・反映
- **バリデーション**: 入力値検証とエラー表示

#### 2.2 AppSettings コンポーネント  
- **アプリ設定表示**: テーマ、言語、通知等の設定項目表示
- **設定変更**: 各種アプリケーション設定の変更
- **保存・適用**: 設定変更の即座反映
- **リセット機能**: デフォルト設定への復元機能

### 3. UI/UX要件

#### 3.1 スタイリングルール準拠
- Tailwind CSS `text-text-*`, `bg-bg-*` パターンの使用
- ダークモード・ライトモード対応の維持
- レスポンシブ対応の継続

#### 3.2 data-testid 属性追加
- `profile-avatar` (プロフィールアバター表示)
- E2Eテスト対応のためのテスタビリティ向上

#### 3.3 ユーザビリティ
- フォーム操作の直感性確保
- 保存状態の明確な可視化
- エラー状態の分かりやすい表示

### 4. 技術要件

#### 4.1 型安全性
- `@/lib/db/schema` からの型定義使用
- スキーマベース型定義への統一
- `@ts-ignore` 系コメント禁止

#### 4.2 Import パス更新
- 移行先ディレクトリからのimport文更新
- アーキテクチャルール準拠のimport グループ化
- 循環依存の回避

#### 4.3 MSWモックシステム連携
- 開発環境でのAPIモック動作確認
- エラー時にMockデータを返さない設計

### 5. テスト要件

#### 5.1 既存テストの移行
- テストファイルも同構造で移行
- テスト内のimport パス更新
- モック設定の整合性確認

#### 5.2 機能テスト
- プロフィール設定の保存・取得動作
- アプリ設定変更の反映確認
- バリデーション機能の動作確認
- エラーハンドリングの適切性

### 6. パフォーマンス要件

#### 6.1 バンドルサイズ
- 移行前後でのバンドルサイズ維持
- Tree shaking の効果確認
- 不要な依存関係の削除

#### 6.2 ランタイムパフォーマンス
- コンポーネントレンダリング性能の維持
- メモ化戦略の適切性

### 7. 品質要件

#### 7.1 静的解析
- TypeScript エラー 0個
- ESLint エラー・警告 0個
- アーキテクチャルール違反 0個

#### 7.2 動的テスト
- 全ユニットテストの pass
- 設定ページでの正常動作確認
- E2Eテストでの主要フロー確認

## 受け入れ基準

### ✅ 完了条件
1. **移行完了**: 全ユーザー関連コンポーネントが `src/model/user/` に配置されている
2. **動作確認**: 設定ページが正常に動作している  
3. **型安全性**: TypeScript エラーが存在しない
4. **品質基準**: ESLint、テストが全て pass している
5. **アーキテクチャ**: 依存関係ルールに準拠している
6. **テスタビリティ**: data-testid属性が適切に追加されている

### 🚫 失敗条件
- 設定機能に不具合が発生している
- 既存のUI/UXが損なわれている  
- パフォーマンスが劣化している
- アーキテクチャルール違反が存在する
