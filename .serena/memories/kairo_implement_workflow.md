# Kairo-Implement ワークフロー

## 概要
kairo-implement は分割されたタスクを順番に、またはユーザが指定したタスクを実装するコマンドです。既存のTDDコマンドを活用して品質の高い実装を行います。

## 実行フロー

### 1. タスク選択・準備
- タスク一覧の確認: `docs/tasks/routine_record-tasks.md`
- 依存関係の確認
- 実装ディレクトリの準備

### 2. 実装タイプ判定

#### A. TDDプロセス（コード実装タスク）
新しいコンポーネント、サービス、フック等の実装

**ステップ:**
1. **要件定義** (`tdd-requirements.md`)
2. **テストケース作成** (`tdd-testcases.md`) 
3. **テスト実装** (`tdd-red.md`)
4. **最小実装** (`tdd-green.md`)
5. **リファクタリング** (`tdd-refactor.md`)
6. **品質確認** (`tdd-verify-complete.md`)

#### B. 直接作業プロセス（準備作業タスク）
プロジェクト初期化、環境構築、設定ファイル作成等

**ステップ:**
1. **準備作業実行**
2. **作業結果確認**

### 3. タスク完了処理
- タスクステータス更新（チェックボックスにチェック）
- 実装結果ドキュメント化
- 次のタスク提案

## 実装タイプ判定基準

### TDDプロセス対象
- 新しいコンポーネント、サービス、フック等の実装
- 既存コードの機能追加・修正
- ビジネスロジックの実装
- API実装

### 直接作業プロセス対象
- プロジェクト初期化・環境構築
- ディレクトリ構造作成
- 設定ファイル作成・更新
- 依存関係のインストール
- ツール設定・設定

## タスク進捗管理

### 現在の状況（例）
- **全タスク数**: 47
- **推定作業時間**: 120-150時間
- **完了済み**: TASK-001, TASK-002 
- **次のタスク**: TASK-003以降

### タスクファイル構成
```markdown
#### TASK-XXX: タスク名

- [ ] **タスク完了**
- **タスクタイプ**: TDD/DIRECT
- **要件リンク**: REQ-XXX
- **依存タスク**: TASK-YYY
- **実装詳細**: ...
- **完了条件**: ...
```

## 品質保証

### TDDプロセス時
- テストファースト開発
- インクリメンタル実装
- 継続的品質確認

### 完了時チェック
1. `npm run type-check`
2. `npm run lint:strict` 
3. `npm run format:check`
4. `npm test`
5. E2Eテスト（必要に応じて）

## エラーハンドリング
- 依存タスク未完了時の警告
- テスト失敗時の詳細表示
- ファイル競合時のバックアップ
