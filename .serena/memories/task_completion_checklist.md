# タスク完了時のチェックリスト

## 必須実行コマンド

### 1. 型チェック
```bash
npm run type-check
```
- TypeScriptコンパイルエラーがないことを確認
- 型定義の整合性チェック

### 2. 厳密なリント
```bash
npm run lint:strict
```
- ESLint警告が0件であることを確認
- コード品質基準を満たしていることを確認

### 3. フォーマットチェック
```bash
npm run format:check
```
- Prettierフォーマット規則に従っていることを確認
- 必要に応じて `npm run format` で自動修正

### 4. ユニットテスト
```bash
npm test
```
- 全ユニットテストが成功することを確認
- 新規追加したコードのテストが含まれていることを確認

### 5. 統合品質チェック
```bash
npm run quality
```
- 上記1-3を一括実行
- 全ての品質基準をクリアしていることを確認

## 条件付き実行コマンド

### E2Eテスト（UIに影響がある場合）
```bash
npm run test:e2e
```
- ユーザーフローに影響する変更の場合に実行
- 重要な機能の回帰テストとして実行

### データベーススキーマ変更の場合
```bash
npm run db:generate  # スキーマ変更後
npm run db:push      # 開発環境への適用
```

## Git関連

### コミット前
```bash
git status           # 変更ファイル確認
git diff            # 変更内容確認
git add .           # ステージング
```

### コミットメッセージ規約
- prefix: feat, fix, refactor, test, docs, style, chore
- 日本語または英語で簡潔に記述
- 例: `feat: ユーザー認証機能の実装`
